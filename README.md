# Paper/Article Summarizer CLI — Structured Summary Generator

Summarizes research papers and articles from URLs or arXiv IDs into structured format.

**Purpose:** Generates structured summaries (key findings, methodology, implications) for research papers and articles.

---

## What It Does

**Core Features:**

1. **arXiv Paper Summarization**
   - Fetches paper metadata from arXiv API
   - Extracts title, authors, abstract, publication date
   - Generates structured summary with key findings, methodology, implications

2. **Structured Output Format**
   - Title, authors, publication date
   - Full abstract
   - Key findings (first 3 sentences of abstract)
   - Methodology (keyword detection from abstract)
   - Implications (last sentence of abstract)
   - Direct links to arXiv and PDF

3. **Batch Processing**
   - Summarize multiple papers from a file
   - One URL/ID per line
   - Process all papers sequentially

4. **Multiple Output Formats**
   - Markdown (default, readable)
   - JSON (programmatic use, API integration)

5. **No API Keys Required**
   - Uses arXiv's public API (free, no authentication)
   - Standard library only (urllib, xml.etree, re)

## Usage

### Summarize arXiv Paper by URL

```bash
paper-summarizer summarize https://arxiv.org/abs/2301.07041
```

Output:
```

# Attention Is All You Need

**arXiv ID:** 2301.07041
**Published:** 2023-01-27T00:00:00Z
**URL:** http://arxiv.org/abs/2301.07041
**PDF:** http://arxiv.org/pdf/2301.07041.pdf

---

## 📝 Abstract

This paper introduces a new neural network architecture called Transformer...

---

## 🔑 Key Findings

This paper introduces a new neural network architecture called Transformer, which relies entirely on self-attention mechanisms. We demonstrate that this architecture achieves state-of-the-art results on machine translation tasks.
```

### Summarize arXiv Paper by ID

```bash
paper-summarizer arxiv 2301.07041
```

Output: Same as above (uses arXiv ID instead of URL)

```bash
paper-summarizer arxiv 2301.07041 --json
```

Output:

### Batch Processing

Create a file `papers.txt` with one arXiv URL or ID per line:

Run batch processing:
```bash
paper-summarizer batch papers.txt
```

## Features

### 1. arXiv API Integration

**Metadata Extracted:**
- Title
- Authors (up to 5)
- Abstract (full text)
- Publication date
- arXiv URL
- PDF URL

**API Used:**
- arXiv Query API (http://export.arxiv.org/api/query)
- No API keys required
- Free, open access

### 2. Structured Summary Generation

**Key Findings:**
- First 3 sentences of abstract
- Captures main contributions
- Easy to scan

**Methodology:**
- Keyword detection from abstract
- Searches for: method, approach, technique, algorithm, framework, model
- Reports if methodology is explicitly stated

**Implications:**
- Last sentence of abstract
- Often contains implications or future work
- Concise summary of impact

### 3. Batch Processing

- Process multiple papers from file
- One URL/ID per line
- Sequential processing (no rate limiting)
- Skip non-arXiv URLs with warning

### 4. Output Formats

**Markdown (default):**
- Human-readable
- Structured with sections
- Direct links to arXiv and PDF

**JSON:**
- Machine-readable
- Programmatic use
- API integration
- Full metadata included

## Architecture

### Summarization Algorithm

```
1. Fetch paper metadata from arXiv API
2. Extract: title, authors, abstract, published date
3. Generate key findings: First 3 sentences of abstract
4. Detect methodology: Keyword matching (method, approach, technique, etc.)
5. Extract implications: Last sentence of abstract
6. Format output: Markdown or JSON
```

### arXiv ID Format

```
Format: YYYY.NNNNN (e.g., 2301.07041)

Where:
- YYYY: Year (2023)
- NNNNN: Sequential ID (07041)
```

## Troubleshooting

### Invalid arXiv ID Format

```bash
# Error: Invalid arXiv ID format. Use: YYYY.NNNNN (e.g., 2301.07041)

# Correct format
paper-summarizer arxiv 2301.07041  # ✅
paper-summarizer arxiv 230107041      # ❌ (missing dot)
paper-summarizer arxiv 23.07041        # ❌ (wrong year format)
```

### Paper Not Found

```bash
# Error: Paper not found: 9999.99999

# Check if arXiv ID is correct
# Visit: https://arxiv.org/abs/9999.99999
# Verify ID format and existence
```

### Non-arXiv URLs

```bash
# Warning: Non-arXiv URLs not yet supported
# Use: 'paper-summarizer arxiv <arxiv-id>' for arXiv papers

# Example of supported URLs
https://arxiv.org/abs/2301.07041  # ✅
http://arxiv.org/abs/2301.07041   # ✅
https://example.com/paper.pdf          # ❌ (not supported yet)
```

### File Not Found

```bash
# Error: File not found: papers.txt

# Create file first
cat > papers.txt << EOF
https://arxiv.org/abs/2301.07041
2301.07041
EOF
```

## Technical Details

### Dependencies

**Python Standard Library Only:**
- `urllib` — Fetch arXiv API
- `xml.etree.ElementTree` — Parse arXiv XML response
- `re` — Extract arXiv IDs from URLs
- `json` — JSON output format
- `argparse` — Command-line interface
- `typing` — Type hints

### arXiv API

**Endpoint:** http://export.arxiv.org/api/query

**Query Parameters:**
- `id_list` — arXiv ID to fetch
- `max_results` — Number of results (default: 1)

**Response Format:** Atom XML (standard syndication format)
